# STATUS — 2026-07-11 — Steamy Bites marketing site v1

## What exists

New Next.js 16 (App Router, TypeScript, Tailwind v4) marketing site in `site/`, separate from
the Vite ordering app in `client/`. Built to the approved **Supper Club** direction
(dark luxury editorial — Fraunces + Instrument Sans, gold `#C9A96A`, arch motif) with a
**Night (default) / Day** theme toggle. Spec: `docs/superpowers/specs/2026-07-11-steamy-bites-site-design.md`.

## Verified working (browser walkthrough, production build)

- **Hero** — footage (`public/media/1.mp4`) clipped inside the gold hairline arch, GSAP
  entrance, steam wisps + R3F particle layer (code-split, unmounts off-screen), watermark
  cropped via overscan, scrim keeps headline legible. Clip lives on the *wrapper* so the
  arch stays aligned (clip on the video itself would inherit the overscan transform).
- **Scroll story** — GSAP pinned, 3 beats cross-fade over footage; last beat stays visible
  (no fade-to-black tail).
- **Menu** — live from `steamybitesbackend.onrender.com/api/menu` with ISR (1h) +
  25s timeout + committed fallback `data/menu-fallback.json` (Render free tier cold-starts).
  Only Cloudinary `dhc55vkds` images allowed through; hover floats an arch-shaped dish photo.
- **Showcase** — editorial grid of macro footage + brand stills, hover zoom, scroll reveals.
- **Location** — stylized SVG map, gold route Laxmi Nagar Metro → cafe animated on scroll,
  Google Maps deep link.
- **Footer** — "See you after dark." arch reprise.
- **Both themes** on desktop + 400px mobile; toggle persists via localStorage, reachable on
  mobile; `prefers-reduced-motion` disables Lenis/pins/particles; custom gold-ring cursor
  (fine pointers only).

## Build & run

```bash
cd site && npm run build && npm run start   # production
cd site && npm run dev                      # dev
```

## Env

- `NEXT_PUBLIC_ORDER_URL` — where "Order Now", the nav "Order" link, and every menu row point
  (all `target="_blank"` into the ordering app; rows show an "order ↗" affordance on hover and
  carry aria-labels). Unset → rows are plain divs and Order Now falls back to `#menu`.
  `.env.local` points at the local ordering app (`cd client && npm run dev -- --port 5180
  --strictPort` — pinned because other Vite projects grab 5173). **Production: set to the
  ordering app's deployed URL and rebuild** (NEXT_PUBLIC_* is inlined at build time).
  The ordering client has no URL routing (in-memory page state), so links land on its home.
- `NEXT_PUBLIC_SITE_URL` — canonical base for OG metadata (defaults localhost).
